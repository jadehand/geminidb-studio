# 更新日志

## 0.3.9

- 修复安装后的桌面客户端使用配置路径而非文件名启动 Sidecar，导致 Windows 返回 `os error 3` 的问题；
- 增加 Tauri `externalBin` 与 Rust Sidecar 启动名称一致性测试，防止安装包再次生成不可用的 Bridge 路径。

## 0.3.8

- 查询编辑器支持悬停查看 UTC、带时区时间以及秒/毫秒时间戳对应的北京时间；
- 时间识别只在明确的时间表达式中生效，不会误判 `LIMIT` 数字或 Measurement 名称；
- 无效日期不显示转换结果，悬停卡片保持单一北京时间输出；
- 固定 Monaco 行号显示、最小位数和装饰区宽度，降低 Windows 缩放下行号裁切风险。
- 修复 Rust 新版借用检查下 Bridge 状态读取的临时锁生命周期错误，恢复 Windows 安装包构建。

## 0.3.7

- 查询工具栏在窄窗口中保留“执行”，将诊断与收藏自动收进“更多”菜单；
- 结果工具栏在窄窗口中保留“复制”，将 CSV、Excel 和 JSON 收进“导出”菜单；
- 去除工作区固定最小宽度，非全屏窗口不再把右侧操作按钮挤出视口；
- 查询结果增加明显的横向滚动提示、键盘焦点和稳定滚动条；
- 分页栏在窄窗口自动分行，页大小和首页、上下页、末页始终可见。

## 0.3.6

- 重构时间转换工具，支持时间戳、北京时间与 UTC 零时区双向转换；
- 自动识别秒、毫秒、微秒和纳秒时间戳，并统一输出北京时间、RFC3339 UTC、秒和毫秒；
- 新增输入时区选择、回车转换、单项复制及 InfluxQL 时间复制；
- 精简结果布局，并说明 GeminiDB Influx 可使用相对时间、RFC3339 或带精度单位的 Unix 时间戳。

## 0.3.5

- 数据目录侧栏支持拖动分隔线调整宽度，范围为 260～620px，并自动适配当前窗口；
- 侧栏宽度自动保存，客户端重启或收起后再次展开仍保持用户设置；
- 分隔线支持左右方向键微调、Home/End 调至边界、双击恢复默认宽度；
- Measurement、天表和 Database 长名称保持单行，超出宽度时可横向滚动；
- 长名称增加完整悬浮提示，避免必须扩大整个客户端窗口才能查看表名。

## 0.3.4

- GeminiDB Influx 连接协议改为显式选择 HTTP 或 HTTPS，新建连接默认使用 HTTP；
- 协议切换只替换地址前缀，保留主机、端口和路径，已保存连接继续使用原协议；
- HTTPS 连接到明文 HTTP 端口时，将 OpenSSL `wrong version number` 转换为“请切换 HTTP”的可读提示；
- “忽略 TLS 证书校验”仅在 HTTPS 模式显示，HTTP 测试、保存和连接时强制关闭该选项；
- 增加 HTTP/HTTPS 地址规范化、协议切换和错误提示测试。

## 0.3.3

- Bridge 启动失败不再终止桌面客户端，主界面始终可以正常打开；
- 增加 Bridge 运行状态、错误原因、本地日志路径和一键重试；
- Bridge 标准错误与退出状态写入本地 `bridge.log`，便于排查安全软件拦截和 Sidecar 路径问题；
- 修复 Bridge 重启时旧进程退出事件覆盖新进程状态的竞态问题；
- GitHub Actions 升级至 Node.js 24 运行时兼容的 `checkout@v7` 与 `setup-node@v7`。

## 0.3.2

- Windows Release 主程序改用 GUI 子系统，启动客户端时不再弹出黑色终端；
- Windows Bridge Sidecar 构建后自动改为 GUI 子系统，避免后台服务额外弹出控制台；
- 增加 PE 文件格式与子系统字段自动化校验，构建异常时直接失败而不是生成错误安装包。

## 0.3.1

- Bridge 生命周期改由 Rust 主进程统一管理，WebView 不再直接启动或销毁 Sidecar；
- Windows 退出时按准确 PID 终止 Bridge 进程树，避免安装升级时文件被残留进程占用；
- Bridge 增加父进程存活监测，桌面主进程异常退出后自动关闭；
- 移除 WebView 的 Shell 启动与终止权限，减少重复启动和权限面。

## 0.3.0

- 查询页签、Database、Measurement 和界面状态自动恢复；
- 异常退出检测、恢复/重新开始入口及最近三份工作区快照；
- 查询结果默认分页并支持 50～1000 行分页数量调整；
- 分页范围、首页/末页及页码跳转，切换查询后自动回到第一页；
- 图表结果页签占位，为后续时序可视化保留稳定入口；
- 本地 Claude CLI 与 Anthropic API 双诊断通道；
- 统一 DiagnosticProvider、错误分类、请求取消和旧结果覆盖保护；
- GeminiDB InfluxQL 本地规则检查、Schema 上下文诊断及性能建议；
- 修复 SQL 逐行差异预览，只允许人工替换或新页签打开；
- Claude API Key 使用系统凭据库存储，诊断默认不发送查询结果与数据库凭据。

## 0.2.0

- Measurement Field、Tag Schema 自动补全与缓存；
- 天表日期筛选及多天表 InfluxQL 生成；
- 结果搜索、排序、列隐藏、固定首列、列宽调整和 Excel 导出；
- 连接复制、删除、环境标识和系统凭据保存；
- 左侧连接与数据目录工具窗视觉精修；
- Git Tag 自动构建并发布 Windows MSI、NSIS 安装包。
