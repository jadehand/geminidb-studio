# 更新日志

## 0.4.3

- 查询结果的 `time` 毫秒时间戳默认以标准 UTC 时间显示；
- 鼠标悬停 `time` 单元格可同时查看北京时间和原始毫秒时间戳；
- 原始查询结果和 CSV、Excel、JSON 导出值保持不变，避免展示格式影响数据精度；
- 查询结果导出增加统一目录设置，默认使用系统下载目录，可选择并记住自定义目录；
- 桌面端导出完成后显示实际保存路径，目录无效或无写入权限时返回明确错误；
- 版本序列由误用的 `4.0.x` 纠正为连续的 `0.4.x`。

## 0.4.2

- 根治 SQL 补全候选框不出现：将 Monaco Suggest 控制器显式打入前端构建，不再只有补全数据却缺少弹层运行模块；
- 自动输入、`Ctrl/Cmd + Space` 与“显示补全”按钮统一使用 Monaco 原生 Suggest Action；
- 改用当前 Monaco 版本支持的真实键盘事件监听，输入关键词前缀、空格或引号时稳定触发上下文补全；
- 补充补全模块装载与触发链回归测试；
- 使用真实 Firefox 验证输入 `select` 会显示 `SELECT` 和查询模板，`Ctrl + Space` 可关闭后重新唤起。

## 0.4.1

- 移除执行记录整行的单击跳转，点击时间、耗时或执行结果不再切换页面；
- 成功、失败信息和 SQL 文本支持正常框选复制；
- 仅双击“命令语句”单元格时，才把该 SQL 放入当前查询窗口；
- 双击恢复 SQL 后保留执行记录页面，并显示轻量成功提示；
- 执行记录顶部增加操作说明，明确单击只查看、双击恢复 SQL；
- 优化长错误信息的换行、浅色/深色主题和窄窗口布局；
- 新增执行记录交互回归测试，防止整行单击行为再次出现。

## 0.4.0

- 新增轻量 InfluxQL 上下文引擎，按光标所在的 SELECT、FROM、WHERE、GROUP BY、ORDER BY 等位置提供不同候选；
- 输入关键词、Field 或 Measurement 前缀时自动弹出补全，不再只依赖空格、双引号或快捷键；
- 编辑器会从当前 SQL 的 FROM 自动识别 Measurement，并按需读取对应 Field、Tag 与类型；
- Schema 请求增加连接/Database/Measurement 级缓存和并发请求合并，避免重复访问 GeminiDB；
- 每个查询标签使用独立 Monaco 模型上下文，避免多标签之间串用 Measurement 或 Schema；
- 补全候选明确区分 Measurement、Field、Tag Key、函数、运算符与 InfluxQL 关键词；
- 函数补全使用可跳转参数片段，并新增 SELECT 查询模板和 WRITE Line Protocol 模板；
- WHERE 增加时间列与常用条件运算符，GROUP BY 增加 time() 与 fill()，ORDER BY 提供时间正序/倒序；
- 编辑器底部显示 Schema 正在读取、已就绪、失败或待选择状态，并适配浅色与深色主题；
- 新增补全上下文回归测试和完整 Web 测试命令。
